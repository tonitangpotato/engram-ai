### Verdict: ✅ PROCEED TO T26C — quality is production-ready, with one bump

### Strengths

1. **Predicate vocab is genuinely meaningful, not LLM filler**
   - 9 distinct predicates, all semantically distinct: `part_of` (21%), `related_to` (18%), `leads_to` (14%), `implements` (14%), `uses` (12%), `is_a` (11%), `depends_on` (5%), `contradicts` (3%), `caused_by` (2%).
   - No `is`, `has`, `the`, or other LLM-template artifacts.
   - The long tail (`contradicts`, `caused_by`) shows the model actually using the contrastive/causal predicates when the source content warrants it — not just defaulting to `related_to`.

2. **Confidence calibration looks reasonable**
   - 0% below 0.50 (driver dropped none; calibration is genuine, not threshold-pumped)
   - 0.5% in 0.50–0.69 (rare hedged extractions, e.g. `用户 --related_to--> 方案可行性` from a 7-character Chinese opinion)
   - 15.4% in 0.70–0.79
   - 69.4% in 0.80–0.89 (modal bucket)
   - 14.6% at 0.90+
   - Distribution shows the model isn't ceiling-pinning confidence. Mean 0.82 with a real left tail.

3. **Subject/object spans are mostly concrete**
   - 95.3% of triples have both spans ≤40 chars (typical concrete nominal: "MVP", "web dashboard", "ISS-024", "Hebbian links").
   - Only **3/363 triples (0.8%)** have whole-sentence spans (>50 chars on either side). All 3 are still semantically valid — just verbose:
     - `refilling old memories with dimensions metadata --leads_to--> enabling old memories to benefit from Dimension Matching mechanism` (cause/effect captured, but spans rephrase the source sentence)
     - `task A6.8 --leads_to--> A6 paragraph added with DONE/files/tests/issue status/notes sections`
     - `hardcoded self.namespace filtering --related_to--> list_topics(alpha) finding topics but get_topic(id) dropping them`
   - These are acceptable. Tolerance threshold for T26c: if >3% trip this, regress to a stricter prompt. Current rate is 27× below that.
   - 0 self-referential triples (subject == object).

4. **CJK handling works** — 17/363 (4.7%) Chinese-character triples. Spot check:
   - `engram 项目 --related_to--> ISS-004` (clean, correct)
   - `LSP --leads_to--> precise call/reference edges` (mixed-language source, clean English triple)
   - `助手 --uses--> 分析` (slightly abstract but defensible)
   - No mojibake, no fragmentation.

5. **Per-memory triple density is healthy**
   - 3.63 triples/memory average.
   - 97/100 produced ≥1 triple (3 zero-triple memories were presumably very short content like "User decided X" — not a quality issue).
   - Memories with structured content (e.g. `00038d05`: "7 task nodes created: cluster-scaling, cs-vptree, cs-storage…") correctly yielded 9 triples — the extractor enumerates list items.

### Concerns (none blocking)

1. **🟡 Tautological `is_a` triples (~3% of total)** — 12 triples follow the pattern `<thing> --is_a--> task_node` / `--is_a--> commit` / `--is_a--> component`. These are correct given the source ("X task nodes created: A, B, C") but don't add retrieval signal beyond what the column already says. **Not worth blocking on** — they're factually correct, just low-value. Could be filtered in a post-process if KC starts over-clustering on these.

2. **🟡 Some abstract subjects in opinion-type memories** — `用户 --related_to--> 助手` from a meta-memory about agent self-reflection. Triples are technically valid but recall would be weird. Source was an emotional/relational memory — these are inherently fuzzy. **Acceptable** as long as the retrieval scorer downweights "用户/助手/User"-as-subject in non-relational namespaces, which is a v0.5 concern, not v0.4.

3. **🟢 Cost extrapolation is fine** — \$14.87 for full 14,871 production memories. At Max plan with personal usage, this is rounding error.

4. **🔴 Wall-clock is the actual constraint, not cost** — 7h at 5 rps. **Recommendation**: T26c should run at `--rps 15` (3× higher). The Anthropic OAuth rate limit on Max is in the 50-RPM range per model, so 15 rps is comfortably safe (≈ 900 RPM). Wall-clock then drops to ~2.5h. If you want it under 1h, push to 20-25 rps but watch for 429s.
