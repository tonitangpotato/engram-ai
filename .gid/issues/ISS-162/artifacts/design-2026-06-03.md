# ISS-162 ExtractionContext — design (draft, 2026-06-03)

> Status: DRAFT pending isolation-sweep result (ISS162-{C,D}-conv26-20260603T171256Z).
> The window-size default and gating posture below are provisional and will be
> finalised once the sweep settles whether the +11.85pp lift is window-size-driven
> or envelope-confounded.

## Problem (recap)

The extractor receives one conversation turn at a time. Bare reply turns
("Researching adoption agencies", "Luna and Oliver! they're so sweet") lose
their coref referent at extraction time, so the self-contained gold fact is
never stored. Shape-scan (2026-06-03): 14/14 conv-26 SEMANTIC-GAP gold turns
are coref-dependent. Proven at bench-driver layer: sliding-window ingest
(window=4) lifts conv-26 overall J 0.2697 → 0.3882 (+11.85pp).

## Why not ISS-178's design

ISS-178 added `prev: Option<String>` (a SINGLE preceding turn) plus a separate
`extract_with_context` trait method. It was falsified (−1.97pp). Two reasons it
differs from the proven window=4 fix:

1. **Window of 1 is too small.** The referent for a bare reply is usually the
   question turn PLUS the answer-setup turn (2-3 turns back). One turn often
   doesn't carry the noun phrase.
2. **Dual-method trait surface.** `extract` + `extract_with_context` with
   fall-through logic is awkward and duplicates retry/parse code per backend.

ISS-178's "do NOT extract from prev" guard wording was NOT the problem — this
design keeps the same quarantine instruction.

## Design

### Type: bounded context window

No new public type needed for the extractor. The context is simply
`Vec<String>` (oldest-first preceding turns) carried on `StorageMeta.context`.
Callers maintain a bounded ring via `TurnWindow` (below). Keeping it a plain
`Vec<String>` (not a wrapper struct) avoids an extra public type and keeps the
serde surface minimal.

### Trait: UNCHANGED (decision revised 2026-06-03 after blast-radius audit)

The first design draft added `context: &ExtractionContext` as a third `extract`
param. **Rejected** — the trait has 16 impls (3 production + ~13 test stubs);
a signature change forces touching all of them = large churn for no benefit,
exactly the "while I'm here" growth karpathy-guidelines warns against.

**Revised:** keep `extract(&self, text, reference)` UNCHANGED. Do context
assembly at the INGEST call site (`store_raw`), prepending the context preamble
to `content` before calling `extract` — exactly what the proven bench-driver
fix does. The extractor stays oblivious; windowing lives in the ingest layer
where it belongs. Zero trait churn, zero impl updates, strictly lower risk.

### Prompt assembly (free helper in memory.rs / store layer)

```rust
fn context_preamble(preceding: &[String]) -> String {
    if preceding.is_empty() { return String::new(); }
    format!(
        "Prior conversation context (for coreference resolution ONLY — do NOT \
extract facts from these lines; they are already stored):\n{}\n\n\
Extract facts ONLY from the turn below, resolving any pronouns or references \
against the context above so each core_fact is self-contained.\n\n",
        preceding.join("\n")
    )
}
```

Final prompt assembly happens in `store_raw`: it prepends
`context_preamble(&meta.context)` to `content`, then calls the UNCHANGED
`extract(assembled, meta.occurred_at)`. `reference_preamble` is still added by
the extractor internally (ISS-190). Order: extractor builds
`EXTRACTION_PROMPT + reference_preamble + <assembled content>`, where
`<assembled content> = context_preamble + original content`. Matches the proven
bench-driver framing.

### Threading through ingest

- Add `StorageMeta.context: Vec<String>` (default empty, serde-skipped).
- `store_raw` (memory.rs:3603 call site): if `meta.context` non-empty, build
  `format!("{}{}", context_preamble(&meta.context), content)` and pass THAT to
  `extract`; else pass `content` unchanged (byte-identical fast path).
- New public ingest entry point for conversational callers:
  `ingest_turn(&mut self, text, occurred_at, context: Vec<String>)` → builds
  `StorageMeta { occurred_at, context, ..default }` and calls `store_raw`.
- `ingest_with_stats_at` stays as-is (empty context) — back-compat.

### Session ring (caller-side)

The library does NOT own conversation state — callers do. A small helper:

```rust
/// Bounded oldest-first ring of recent turns for building ExtractionContext.
pub struct TurnWindow { cap: usize, turns: VecDeque<String> }
impl TurnWindow {
    pub fn new(cap: usize) -> Self { ... }
    /// Snapshot current window as context, THEN push the new turn.
    pub fn context_then_push(&mut self, turn: &str) -> Vec<String> { ... }
}
```

Bench driver replaces its inline window logic with `TurnWindow`. Production
conversational channels (Telegram/Discord ingest) get the same helper.

## Window size

Provisional default **N=4** (the proven value). Final value pinned by the
isolation sweep:
- If window=1 (arm C) also lifts → smaller default is fine, pick min that holds.
- If window=1 flat but window=4 lifts → N≥3 required; default 4.

## Double-extraction guard

Overlapping windows could re-extract a fact already stored from an earlier
turn. Mitigations, in order:
1. **Prompt quarantine** ("do NOT extract from context") — primary; verified
   4/4 in `iss201_window_verify` (context turns NOT re-extracted).
2. **0.95 dedup merge** — backstop for near-identical facts that slip through.

## Tests

- `context_empty_is_byte_identical` — assembled prompt == legacy when context empty.
- `context_preamble_quarantines` — preamble contains the do-not-extract guard.
- `turn_window_bounded` — ring never exceeds cap, oldest evicted, snapshot-before-push.
- `ingest_turn_threads_context` — end-to-end: a bare reply turn + context yields a
  self-contained core_fact (uses a stub extractor that echoes the assembled prompt).
- LLM-level: reuse `iss201_window_verify` against the library path (not just bench).

## Migration

- **No trait change** → zero external `MemoryExtractor` impl churn (the first
  draft's 3-param trait would have forced touching 16 impls; rejected).
- `StorageMeta` gains one optional field (`context: Vec<String>`, default empty,
  serde-skipped) — additive, non-breaking for existing `store_raw` callers.
- Bench driver: swap inline window for `TurnWindow` + `ingest_turn`; the
  `ENGRAM_BENCH_INGEST_WINDOW` flag stays as the A/B harness, now driving the
  library path so the +11.85pp baseline validates the real implementation.
