## Findings

1. **ISS-068 fix verified at the data layer.** 58/58 turns admitted; all 3 cited gold turns are in the substrate.

2. **ISS-068 fix did NOT improve hit@5 — it slightly regressed it.** The 19 newly-admitted turns enlarged the candidate pool, and 2 of them (q14's D2:4 contender, q22's contenders) ranked higher than the true gold turn for unrelated queries. This is candidate-pool-induced ranking churn, not a real retrieval improvement or regression.

3. **Retrieval ranking is the bottleneck, not substrate completeness.**
   - For q2/q19/q20: query↔content lexical+semantic gap is unbridgeable by current FTS+embedding (e.g. q19 "What does Melanie think about Caroline's adoption?" vs gold "You're doing something amazing! Creating a family for those kids is so lovely" — no shared lemmas, weak embedding bridge through "family/adoption").
   - For q14/q22: gold and distractor turns are semantically very close; ranking is unstable under candidate pool changes.

4. **No instrumentation regression.** Ingestion logs cleanly; 0 silent drops; substrate count matches expectation.

## Conclusions

- ISS-068 is **legitimately resolved** (data integrity), but its retrieval-impact prediction in the resolution note was wrong — the issue's "lifts hit@5 by 3 queries" expectation didn't materialize because the surfacing problem (ranking) is independent of the admission problem (which ISS-068 fixed).
- The retrieval bottleneck is now fully exposed:
  - **Ranking instability** (q14, q22) — needs a tie-breaker / cross-encoder / better blending
  - **Query↔content semantic gap** (q2, q19, q20) — needs query rewriting or graph-walking retrieval
  - Hybrid plan (q13, q21) — still emits 2 empty result sets (unchanged)
- Stagnation guard would now fire (56% → 48%, Δ = −8 pp, but **direction is negative** which means we're not in a productive zone — the right move is to **stop optimizing this metric for now** and address the underlying ranking architecture).
